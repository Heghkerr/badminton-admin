# Dokumen Desain — Badminton Admin

Dokumen ini jadi “blueprint” implementasi untuk aplikasi admin komunitas badminton: **pembayaran**, **kas**, **sesi main**, **pairing otomatis (fairness + grade)**, dan **pencatatan skor 21**.

## Mulai dari sini

- `MVP.md`: versi minimal yang paling cepat dipakai komunitas
- `ERD.dbml`: rancangan tabel/relasi database (bisa dipakai untuk bikin migration)
- `PAIRING.md`: aturan pairing, fairness, dan contoh skenario grade (termasuk “menggendong”)
- `FLOW_SESSION.md`: alur operasional dari check-in sampai game selesai
